# C.L.E.A.R.框架

## 概述

C.L.E.A.R. 框架是对[[提示词六维度万能公式]]的进阶升级。它不仅告诉你"写什么"，还告诉你"怎么想"。从"会用工具"到"建立系统化方法论"的分水岭。

## 五步详解

### C — Context（语境）
在动笔之前，先回答三个问题：
- **使用场景**：电商主图？社交配图？个人作品集？客户交付？
- **目标受众**：年轻女性？科技从业者？高端消费者？
- **输出格式**：1:1 正方形？9:16 竖版？16:9 横版？

语境决定了后续所有维度的选择。

### L — Layer（分层）
按层级组织提示词，从最重要到最次要：

| 层级 | 内容 | 说明 |
|------|------|------|
| 第1层 | 主体层 | 画面的核心是什么 |
| 第2层 | 环境层 | 在什么环境/背景中 |
| 第3层 | 光效层 | 什么光线和色彩 |
| 第4层 | 风格层 | 什么艺术风格/技术参数 |
| 第5层 | 质量层 | 画质要求和排除项 |

各层之间用换行分隔。结构化分层的提示词比"一段话写到底"效果更稳定。

### E — Exclude（排除）
系统化管理排除词，按场景类型分类：

**人像类**：`no plastic skin, no airbrushing, no extra fingers, no deformed hands`
**产品类**：`no watermark, no shadow on white background, no extra objects`
**海报类**：`no misspelled text, no extra text, no blurry typography, verbatim text only`

### A — Adjust（调整）
迭代优化决策树：

```
生成结果 → 分析问题
├── 光线不对 → 修改光线描述
├── 构图不对 → 修改镜头/景别/视角
├── 色调不对 → 修改色彩/风格
├── 主体不对 → 修改主体特征
├── 质感不对 → 修改材质/胶片/后期
└── 多个问题 → 只改最严重的一个
```

**核心原则：每轮只改一个变量。**

### R — Refine（精炼）
最终打磨：
- 删除重复的描述
- 删除互相矛盾的描述
- 删除对效果没有贡献的填充词
- 确认关键信息没有被埋没

一条好的提示词应该像一段好的代码——每个词都有其存在的理由。

## 三大模板设计模式

### 1. 叙事流（Narrative Flow）
按"从大到小、从远到近"顺序描述。适合人像和场景。

```
[总体氛围/风格设定]。[场景环境]。[主体外貌]。[姿态动作]。[光线色彩]。[质量排除]。
```

### 2. 参数流（Parameter Flow）
用 `{参数名}` 占位符构建可复用模板。适合批量生成场景。

```
A {风格} product photography of {产品名称} on {背景}.{光线} lighting from {方向}. {比例} ratio.
```

### 3. 分层流（Layered Flow）
按区域/层级逐条列出。适合 UI、信息图、复杂排版。

```
[顶部区域] Navigation bar: ...
[中部区域] Hero card: ...
[底部区域] Tab bar: ...
```

## 迭代优化示例（咖啡产品图）

| 轮次 | 改动 | 效果 |
|------|------|------|
| 第1轮 | 初始：`A cup of coffee on a table` | 光线平淡，构图随意 |
| 第2轮 | 添加光线方向+材质描述 | 有了质感和氛围 |
| 第3轮 | 添加场景道具+窗外元素 | 有了故事感，专业级 |
